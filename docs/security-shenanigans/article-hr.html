How to Configure a Home SIEM

Wazuh is an open-source SIEM and XDR platform that enables centralized collection, analysis, and correlation of security events across systems.

It is used for log monitoring, detection, and incident correlation in real time.

1. Virtual Machine Setup

  • 4 GB RAM
  • 2 CPU cores
  • 20–40 GB disk space
  • Stable network connection

2. Installing Wazuh Server

curl -sO https://packages.wazuh.com/4.7/wazuh-install.sh
sudo bash wazuh-install.sh -a

3. Adding Agents

Agents collect logs from endpoints and forward them to the Wazuh manager.

4. Ingesting Firewall Logs

/var/ossec/logs/archives/archives.log
sudo systemctl restart wazuh-manager

5. Decoder

Decoders convert raw logs into structured fields.

6. Rules

<rule id="100040" level="3">
  <decoded_as>Firewall_Generic</decoded_as>
  <description>Firewall event</description>
</rule>

7. Log Testing (Required)

Wazuh includes a built-in tool for testing decoders and rules. The documentation states: “It is important that you see the fields in Phase 2 and the alert in Phase 3.”

Run:

/var/ossec/bin/wazuh-logtest

Paste a firewall log (without the timestamp prefix added by rsyslog), for example:

device_name="FW" timestamp="2024-01-01T12:00:00+0100" log_type="Firewall" src_ip="1.2.3.4" dst_ip="5.6.7.8" protocol="TCP" src_port=1234 dst_port=443

If everything is correct:

Phase 1: pre-decoding
Phase 2: extracted fields (decoder works)
Phase 3: rule triggered (alert generated)

Restart the manager:

sudo systemctl restart wazuh-manager

Firewall logs will then appear in the Wazuh dashboard.

Final Thoughts

A home SIEM is not just an educational project — it provides real visibility into security events inside a network.

Wazuh is powerful enough for production use, but simple enough for a lab environment.

With proper configuration, endpoint activity and network traffic can be monitored in real time.
