Kako konfigurirati kućni SIEM

Wazuh je open-source SIEM i XDR platforma koja omogućuje centralizirano prikupljanje, analizu i korelaciju sigurnosnih događaja iz različitih sustava.

Koristi se za monitoring logova, detekciju i korelaciju incidenata u stvarnom vremenu.

1. Priprema virtualnog stroja

  • 4 GB RAM
  • 2 CPU jezgre
  • 20–40 GB disk
  • Stabilna mrežna povezanost

2. Instalacija Wazuh servera

curl -sO https://packages.wazuh.com/4.7/wazuh-install.sh
sudo bash wazuh-install.sh -a

3. Dodavanje agenata

Agenti prikupljaju logove s endpointa i šalju ih na Wazuh manager.

4. Ingestiranje firewall logova

/var/ossec/logs/archives/archives.log
sudo systemctl restart wazuh-manager

5. Decoder

Decoders pretvaraju sirove logove u strukturirane podatke.

6. Pravila

<rule id="100040" level="3">
  <decoded_as>Firewall_Generic</decoded_as>
  <description>Firewall event</description>
</rule>

7. Testiranje logova (obavezno)

Wazuh uključuje ugrađeni alat za testiranje dekodera i pravila. Dokument navodi: “Važno je da vidite polja u Phase 2 i alert u Phase 3.”

Pokreni:

/var/ossec/bin/wazuh-logtest

Zalijepi firewall log (bez timestamp prefiksa koji dodaje rsyslog), primjer: 

device_name="FW" timestamp="2024-01-01T12:00:00+0100" log_type="Firewall" src_ip="1.2.3.4" dst_ip="5.6.7.8" protocol="TCP" src_port=1234 dst_port=443

Ako je sve ispravno:

Phase 1: pre-decoding
Phase 2: extracted fields (decoder radi)
Phase 3: rule triggered (alert generiran)

Restartaj manager:

sudo systemctl restart wazuh-manager

Firewall logovi će se pojaviti u Wazuh dashboardu.

Završna misao

Kućni SIEM nije samo edukativni projekt — omogućuje stvarni uvid u sigurnosne događaje unutar mreže. Wazuh je dovoljno moćan za produkciju, ali i dovoljno jednostavan za lab okruženje.

Uz pravilnu konfiguraciju moguće je pratiti endpoint aktivnosti i mrežni promet u stvarnom vremenu.

U nastavku serije obrađivat će se dodatne konfiguracije i sigurnosni scenariji za izgradnju kompletnog sigurnosnog sustava.
