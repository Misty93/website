How to Configure a Home SIEM

Wazuh is an open-source SIEM and XDR platform that provides centralized collection, analysis, and correlation of security events from endpoints, network devices, and cloud services.

Preparing the Virtual Machine

Recommended minimum resources:

  • 4 GB RAM
  • 2 CPU cores
  • 20–40 GB disk space
  • Stable network connection

Resource consumption scales with the number of agents and generated events.

Installing the Wazuh Server

curl -sO https://packages.wazuh.com/4.7/wazuh-install.sh
sudo bash wazuh-install.sh -a

Once installation finishes, the dashboard is usually available at: 

https://IP-address:5601

Adding Agents

Wazuh agents collect logs from Windows, Linux, and macOS endpoints.

  • Wazuh server IP address
  • Agent key
  • Agent hostname

Firewall Log Ingestion

If logs are not visible, enable archiving inside ossec.conf. 

<global>
  <alerts_log>yes</alerts_log>
  <logall>yes</logall>
  <logall_json>yes</logall_json>
</global>
sudo systemctl restart wazuh-manager

Firewall Decoder Example

<decoder name="Firewall_Generic">
  <type>syslog</type>
  <prematch>device_name="</prematch>
</decoder>

<decoder name="Firewall_Generic_child">
  <parent>Firewall_Generic</parent>
  <regex>
device_name="(\S+)"
timestamp="([^"]+)"
log_type="([^"]+)"
src_ip="([^"]+)"
dst_ip="([^"]+)"
protocol="([^"]+)"
src_port=(\d+)
dst_port=(\d+)
  </regex>

  <order>
device_name,timestamp,log_type,src_ip,dst_ip,protocol,src_port,dst_port
  </order>
</decoder>

Local Rule Example

<group name="custom_firewall">
  <rule id="100040" level="3">
    <decoded_as>Firewall_Generic</decoded_as>
    <description>Firewall Log Event</description>
  </rule>
</group>

Testing the Decoder

/var/ossec/bin/wazuh-logtest
